    \hypertarget{if-statement}{%
\section*{If Statement}\label{if-statement}}

The \texttt{if} statement is used to execute a block of code if a
condition is true. The syntax of an if statement is:

\begin{verbatim}
if condition:
    block of code
\end{verbatim}

where \texttt{condition} must be/evaluate to a boolean value. If
\texttt{condition} evaluates to \texttt{True} then control moves to the
block of code indented after the \texttt{:} and it is executed. If
\texttt{condition} evaluates to \texttt{False}, then the block of code
is skipped and control moves on to the code after the \texttt{if}
statement.





    \hypertarget{worked-example}{%
\subsubsection*{Worked Example}\label{worked-example}}

Let's consider the case where we want to check if one variable is
greater than the other:



    \begin{tcolorbox}[breakable, size=fbox, boxrule=1pt, pad at break*=1mm,colback=cellbackground, colframe=cellborder]
\prompt{In}{incolor}{2}{\boxspacing}
\begin{Verbatim}[commandchars=\\\{\}]
\PY{n}{a} \PY{o}{=} \PY{l+m+mi}{3}
\PY{n}{b} \PY{o}{=} \PY{l+m+mi}{2}

\PY{k}{if} \PY{n}{a} \PY{o}{\PYZgt{}} \PY{n}{b}\PY{p}{:}
    \PY{n+nb}{print}\PY{p}{(}\PY{n}{a}\PY{p}{,} \PY{l+s+s1}{\PYZsq{}}\PY{l+s+s1}{is greater than}\PY{l+s+s1}{\PYZsq{}}\PY{p}{,} \PY{n}{b}\PY{p}{)}
\end{Verbatim}
\end{tcolorbox}

    \begin{Verbatim}[commandchars=\\\{\}]
3 is greater than 2
    \end{Verbatim}



    If we ran the code above but with

\begin{verbatim}
a = 2
b = 2
\end{verbatim}

then we would see nothing printed out.





    \hypertarget{else}{%
\subsection*{Else}\label{else}}

What if you wanted to execute a code block if a statement is true; and
another if it's false. The \texttt{else} part of an if statement can be
used for this:

\begin{verbatim}
if condition:
    code block 1
else:
    code block 2
\end{verbatim}

If \texttt{condition} evaluates to \texttt{True} then
\texttt{code\ block\ 1} will be executed. If, on the other hand,
\texttt{condition} evaluates to \texttt{False}, \texttt{code\ block\ 2}
will be executed.

    \hypertarget{elif}{%
\subsection*{Elif}\label{elif}}

Now, what if you have more than 2 mutually/partially exclusive
conditions? This is a job for \texttt{elif} statements:

\begin{verbatim}
if condition_1:
    code block 1
elif condition_2:
    code block 2
elif condition_3:
    code block 3
\end{verbatim}

Here computer first checks \texttt{condition\_1}. If
\texttt{condition\_1} evaluates \texttt{True} then
\texttt{code\ block\ 1} is executed and control moves leaves the if
statement. If \texttt{condition\_1} evaluates as \texttt{False} then the
computer will check \texttt{condition\_2}, if this evaluates as
\texttt{True} then \texttt{code\ block\ 2} will be executed and control
will leave the if statement. If both \texttt{condition\_1} and
\texttt{condition\_2} are both \texttt{False}, then the computer will
check \texttt{condition\_3}, if this evaluates to \texttt{True}
\texttt{code\ block\ 3} will be executed and control will leave the if
statement.

See the flow chart in the \textbf{\texttt{else} and \texttt{elif}}
section if this explanation was confusing.

An alternative to using \texttt{elif} statements is nested
\texttt{if}/\texttt{else} statements:

\begin{verbatim}
if condition_1:
    code block 1
else:
    if condition_2:
        code block 2
    else:
        if condition_3:
            code block 3
\end{verbatim}

This is quite messy. A general rule of thumb for Python is to avoid
nesting where possible. There are certain scenarios where nested
\texttt{if} statements are desired, however.

Never use multiple \texttt{if} statements to do the job of \texttt{elif}
statements:

\begin{verbatim}
if condition_1:
    code block 1
if (not condition_1) and condition_2:
    code block 2
if (not condition_1) and (not condition_2) and condition_3:
    code block 3
\end{verbatim}

this is not only annoying to write, it is computationally expensive. If
one of these conditions is true, then the others aren't, but the
computer will still check each condition in turn.

    \hypertarget{else-and-elif}{%
\subsubsection*{Else and Elif}\label{else-and-elif}}

If you include an \texttt{else} part of an \texttt{if} statement with
\texttt{elif} parts, the code block in the \texttt{else} statement only
executes if all of the conditions in the \texttt{if} and \texttt{elif}
statements that precede it are false.

\begin{figure}
\centering
\includegraphics{if_elif_else.png}
\caption{if elif else flow chart}
\end{figure}
